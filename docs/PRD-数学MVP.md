# 智考通 · 产品需求文档（数学学科 MVP）

- 版本：v0.1（MVP / 第一阶段 Month 1-4）
- 范围：**仅数学学科**，打通核心链路；物理/化学/语文等在第二阶段扩展。
- 定位：基于纳米级**认知诊断**的个性化提分工具。**不押题、不预测原题、不承诺命中率。**

## 1. 目标与非目标

### 1.1 MVP 目标
1. 跑通端到端链路：解析 → 解题（苏格拉底引导）→ 认知诊断 → 自适应推荐。
2. 数学认知诊断可解释：输出知识点掌握度（4 级）、薄弱点排序、错误归因、能力雷达图。
3. 合规闸门可用：高考熔断、防沉迷全链路生效。
4. 内测：面向 1000 名高三学生，验证"提分效率 + 学习时长"假设。

### 1.2 非目标 / 已超出 MVP 的进展
- ❌ 任何押题/原题预测/押中率功能与宣传（永久红线）。
- ✅ 多学科：已扩展 **数学 + 物理 + 化学 + 生物**（统一图谱 + 跨学科边 + 按学科切片诊断/推荐）。
- ✅ 作文/主观题批改：已落地（rubric 评分，可热插拔为微调模型）。
- ✅ 真实 OCR/大模型接入：生产客户端已落地并经 HTTP 契约层验证（Mock↔real 配置切换）。
- 语文/英语/政史地等学科为后续扩展（框架已就绪，填内容即可）。

## 2. 目标用户与核心场景

| 角色 | 画像 | 核心诉求 |
|---|---|---|
| 高三学生（主） | 数学中等偏下，时间紧，错题反复 | 用最少时间补最该补的短板，会做不只是看会 |
| 家长 | 关注投入产出与用眼/用时 | 看得懂的进步报告、可控的使用时长 |
| 一线教师（B 端） | 班级提分、减轻批改负担 | 班级薄弱点透视、可信的题目质量 |

**核心场景**：学生拍照上传一道不会的数学题 → 系统**不直接给答案**，先问 3 个引导问题 → 学生尝试后逐级展开分步思路与完整解法 → 系统据此更新认知诊断 → 推送最贴合其"最近发展区"的强化/巩固/提高题。

## 3. 功能需求（与代码模块一一对应）

### F1 多模态解析　`P0`　→ [parsing.py](../backend/app/modules/parsing.py)
- 输入：题目文本/图片（MVP 文本+Mock OCR）、可选手写解题步骤。
- 输出：统一 JSON `ParsedProblem`（题型、题干、选项、LaTeX 公式块、挂载知识点、难度、**手写错误步骤标记**）。
- 验收：公式转 LaTeX；选择题选项正确切分；手写步骤能标出错误步与错误类型。✅

### F2 苏格拉底式解题　`P0`　→ [solver.py](../backend/app/modules/rag/solver.py)
- **硬约束**：首次最多到 HINT（仅 3 个引导问题，无答案）；需 HINT→GUIDED→FULL 逐级、不可跳级。
- FULL 级展示**完整思维链 + 答案**；命中题库用标准解法（保证数学正确），自由输入只给方法论引导、不杜撰答案。
- 幻觉抑制：输出知识点对照知识图谱核验。
- 验收：[test_solver_socratic.py](../backend/tests/test_solver_socratic.py) 全绿。✅

### F3 认知诊断　`P0`　→ [diagnosis/engine.py](../backend/app/modules/diagnosis/engine.py)
- 模型：NeuralCD（多知识点静态画像，可解释）+ BKT（掌握随作答演化），加权融合，图传播补全无证据知识点。
- 维度：掌握度 4 级、错误类型 5 类、能力层级 6 级、模块雷达图、薄弱点排序、自然语言解读。
- 冷启动：30 题 / ~15 分钟蓝图（`build_cold_start_test`）。
- 验收：能在 5–8 题内定位到导数/数列等薄弱模块。✅

### F4 自适应推荐　`P1`　→ [recommend.py](../backend/app/modules/recommend.py)
- ZPD：优先预测答对率 0.6–0.8；配比 70/20/10（强化/巩固/提高）；遗忘曲线判定复习到期。
- 触发：每完成 5 题重算。
- 验收：列表以薄弱强化为主，预测答对率合理，不重复刚做过的题。✅

### F5 合规闸门　`P0`　→ [compliance.py](../backend/app/core/compliance.py)
- 高考熔断、防沉迷（时长/宵禁）、家长监护设置。见 [合规设计.md](合规设计.md)。✅

### F6 错题本 + 智能复习排程　`P1`　→ [modules/review/](../backend/app/modules/review/)
- 答错自动入册；熔断期间仍可用。✅
- **智能排程**：SM-2 间隔重复 + 遗忘曲线 + 掌握度调制；每日复习队列按遗忘紧迫度排序；
  毕业/重学状态机；统计与 7 天到期预测。`/review/queue`·`/review/grade`·`/review/stats`·`/review/forecast`。
- 验收：[test_review.py](../backend/tests/test_review.py)（排程正确性、遗忘曲线单调、优先级、毕业、队列/统计）全绿。✅

### F0 AI 提分规划师　`P0`（考生最高频刚需）　→ [modules/planner/](../backend/app/modules/planner/)
- 输入：诊断掌握度 + 高考考点分值蓝图 + 距高考天数 + 每日可用时间 → 输出**按"分/小时"性价比最优**的备考计划。
- 提分性价比排序（高分值 + 低掌握 + 易提分优先）、贪心分配 + 前置打地基、倒计时/熔断感知的每日任务（跨学科交错 + 错题复习）、预计提分。
- 验收：[test_planner.py](../backend/tests/test_planner.py)（权重/ROI 排序/预算/前置/熔断）全绿。✅ 前端 Vue「提分规划」页 + RN「规划」Tab。
- **模考反馈闭环**：录入真实模考分 → 校准估分(纠正系统性高/低估) + 识别「会做却失分」(应试缺口) + 退步趋势 → 回灌规划重排各科优先级、把预计提分锚定真实模考。
  `POST /mock`·`GET /score/{uid}`·`/plan` 自动含反馈。验收 [test_mock_feedback.py](../backend/tests/test_mock_feedback.py) 全绿。✅
- **应试丢分细分到题型 + 限时训练**：录入分题型得分(选择/填空/解答·易中难压轴) → 按掌握度估"该拿多少"定位丢在哪类题、什么原因 → 开出限时训练(选择手滑 8 题/12min、压轴"只做最后一问抢前两步的分" 3 题/25min)，按可挽回分排序、自动排进每天首项。
  `POST /mock`(含 `section_scores`)·`GET /execution/{uid}`·`GET /exam/paper`。验收 [test_execution.py](../backend/tests/test_execution.py) 全绿。✅ 前端 Vue/RN 规划页「按题型录入」。

### F7 变式题生成　`P1`　→ [modules/variant/](../backend/app/modules/variant/)
- **参数化模板 + 符号求解器**：保持考点/数学模型不变，改参数·情境·设问，答案算出保正确。
- 质量控制：规则校验 + 难度预测 + 去重 + 版权相似度(≤0.30) + 内容安全 → **人工审核**(仅 APPROVED 可用)。
- 情境库 10 大类；生成题回流苏格拉底解题/诊断/推荐；高考期间熔断。
- 验收：[test_variant.py](../backend/tests/test_variant.py) 含 6 模板数学正确性独立验证，全绿。✅

### F8 主观题批改　`P2`　→ [modules/grading/](../backend/app/modules/grading/)
- 作文：内容/表达/发展等级 3 维 × 4 二级维度（满分 60），透明细则评分 + 校准；输出各维度得分、优缺点、修改建议、范文指引。
- 文综/理综：采分点语义匹配按点给分 + 逻辑评价 + 改进建议。高考期间熔断。
- 验收：[test_grading.py](../backend/tests/test_grading.py)（好作文>差作文、采分点命中、校准、合规）全绿。✅
- 说明：MVP 用确定性 rubric 评分（可热插拔为 10 万阅卷样本微调模型）。

### Ops MLOps（运维，非学生功能）→ [modules/mlops/](../backend/app/modules/mlops/)
- 模型监控告警（Prometheus + 规则告警）、A/B 灰度发布（金丝雀 + 晋升/回滚）、特征/标签漂移检测（PSI）。
- 与训练流水线闭环：监控/漂移 → 告警 → 增量重训 → 影子评估/灰度 → 达标晋升 → 热加载。

## 4. 非功能需求

| 维度 | 指标（生产目标） | MVP 现状 |
|---|---|---|
| OCR/公式识别 | OCR≥99%、公式≥98% | Mock（接口就绪，待接百度/阿里云）|
| 解题准确率 | ≥95%（GAOKAO-Bench） | 命中题库为标准解，准确；自由输入待接大模型 |
| 系统响应 | ≤2s；高峰可用性≥99.9% | 进程内毫秒级（无外部依赖）|
| 变式题合格率 | ≥90%（教师评审） | 参数化模板+符号求解，数学正确性 100%；情境/难度待教师评审 |
| 诊断颗粒度 | ≤1000 知识点/学科 | 数学 23 点（种子，可扩）|
